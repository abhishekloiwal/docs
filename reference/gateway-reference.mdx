## Endpoints in v1

- `GET /v1/manifest`
- `POST /v1/invoke`
- `GET /v1/jobs/{jobId}`
- `POST /v1/jobs/{jobId}/cancel`

This reference matches the current hosted Atlas contract:

- one workspace gateway
- manifest-driven action discovery
- one action per invoke request
- optional async jobs
- optional approval flow
